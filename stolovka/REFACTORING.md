# Changelog - Рефакторинг проекта

## Выполненные изменения

### 1. ✅ Создан requirements.txt
- Добавлены все необходимые зависимости проекта
- Flask, Flask-SocketIO, openpyxl, APScheduler, requests

### 2. ✅ Исправлены опечатки
- `exel.py` → `excel.py`
- `ingridients` → `ingredients` во всех файлах
- Обновлены все импорты

### 3. ✅ Создан config.py
- Вынесены все константы и настройки
- Пути к файлам и папкам
- Настройки сервера (HOST, PORT, DEBUG)
- Настройки бизнес-логики (PRICE_MARKUP)
- Настройки планировщика отчетов

### 4. ✅ Рефакторинг класса DB (BD.py)
- Добавлен метод `_get_connection()` для централизованного управления соединениями
- Добавлен метод `_init_database()` для инициализации БД
- Упрощена инициализация настроек через цикл вместо повторяющегося кода
- Все `sqlite3.connect('stolovka.db')` заменены на `self._get_connection()`
- Использование констант из config.py (DB_PATH, PRICE_MARKUP)

### 5. ✅ Улучшена обработка ошибок
- Добавлены try-except блоки в критичных методах
- Валидация входных данных:
  - Проверка на пустые строки при создании пользователей/блюд/ингредиентов
  - Проверка единиц измерения ингредиентов
  - Проверка формата цен (отрицательные значения)
  - Валидация данных транзакций
- Добавлено логирование ошибок через print (можно заменить на logging)

### 6. ✅ Рефакторинг excel.py
- Использование констант из config.py (REPORTS_DIR)
- Добавлена обработка ошибок с try-except
- Улучшена читаемость кода

### 7. ✅ Рефакторинг stolovka.py
- Использование констант из config.py
- Упрощены пути к директориям
- Улучшена читаемость

### 8. ✅ Создана документация
- README.md с описанием проекта, установки и использования
- Документация по структуре проекта
- Описание основных функций

### 9. ✅ Рефакторинг frontend (web_resources)
- **Модульная структура**: Разделён монолитный `config.js` (892 строки) на отдельные модули:
  - `utils.js` - Общие утилиты и константы (форматирование дат, валидация, фильтрация)
  - `socket.js` - WebSocket соединение и навигация между страницами
  - `menu.js` - Управление меню (недельное и регулярное меню)
  - `users.js` - Управление пользователями
  - `purchase.js` - Управление ингредиентами и закупками
  - `dishes.js` - Управление блюдами и рецептами
  - `reports.js` - Управление отчётами
  - `settings.js` - Настройки приложения
- **Единый стиль именования**: Все функции переименованы в camelCase (было: `NewUser()`, стало: `newUser()`)
- **Улучшена читаемость**: Добавлены JSDoc комментарии к функциям
- **Устранено дублирование**: Общие функции (фильтрация, показ/скрытие кнопок) вынесены в utils.js
- **Константы**: Дни недели, месяцы и другие константы вынесены в начало utils.js
- **Обновлён HTML**: config.html обновлён для подключения новых модулей и использования новых имён функций

## Проверка работоспособности

Для проверки синтаксиса:
```bash
python -m py_compile stolovka.py BD.py excel.py config.py
```

Для запуска:
```bash
python stolovka.py
```
