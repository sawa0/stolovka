# Frontend структура проекта

## Модульная архитектура

Клиентский JavaScript разделён на отдельные модули по функциональности:

### Основные модули

#### `utils.js` - Утилиты и общие функции
- Константы (дни недели, месяцы)
- Форматирование дат (ISO-недели, месяцы отчётов)
- Валидация данных
- Фильтрация списков
- Вспомогательные функции (padString, createEditButtonHandlers)

#### `socket.js` - WebSocket и навигация
- Инициализация WebSocket соединения
- Обработка событий подключения/переподключения
- Навигация между страницами (`openPage()`)
- Инициализация при загрузке страницы

#### `menu.js` - Управление меню
- Отображение недельного меню
- Управление регулярным меню
- Обновление блюд в меню
- Печать меню
- Подсчёт стоимости дня

#### `users.js` - Управление пользователями
- Список пользователей
- Добавление/редактирование/удаление пользователей
- Активация/деактивация пользователей
- Фильтрация списка

#### `purchase.js` - Управление ингредиентами
- Список ингредиентов
- Добавление/редактирование/удаление ингредиентов
- Редактирование цен
- Фильтрация списка

#### `dishes.js` - Управление блюдами и рецептами
- Список блюд
- Добавление/редактирование/удаление блюд
- Активация/деактивация блюд
- Управление рецептами (добавление/удаление ингредиентов)
- Редактирование объёмов ингредиентов
- Скачивание рецептов
- Фильтрация списка

#### `reports.js` - Управление отчётами
- Месячные отчёты
- Детали отчётов за день
- Фильтрация по пользователям
- Скачивание отчётов
- Удаление заказов/отчётов

#### `settings.js` - Настройки приложения
- Настройки подтверждения заказов
- Настройки Telegram (автоотправка отчётов)
- Тестирование Telegram API
- Обновление приложения

## Соглашения об именовании

### Функции
- **camelCase** для всех функций: `newUser()`, `filterUserList()`, `openPage()`
- Глаголы в начале: `get`, `set`, `update`, `delete`, `create`, `show`, `hide`

### Переменные
- **camelCase** для переменных: `activePage`, `activeReportData`, `dishToDelete`
- Константы в **UPPER_SNAKE_CASE**: `DAYS_OF_WEEK`, `MONTHS_SHORT`

### Обработчики событий
- Префикс по типу действия: `show*`, `hide*`, `edit*`, `delete*`, `update*`
- Суффикс `WithDelay` для отложенных действий

## Порядок подключения модулей в HTML

```html
<!-- Утилиты и общие функции -->
<script src="/utils.js"></script>

<!-- WebSocket и базовая инициализация -->
<script src="/socket.js"></script>

<!-- Модули функциональности -->
<script src="/menu.js"></script>
<script src="/users.js"></script>
<script src="/purchase.js"></script>
<script src="/dishes.js"></script>
<script src="/reports.js"></script>
<script src="/settings.js"></script>
```

**Важно**: `utils.js` должен загружаться первым, так как другие модули используют его функции.

## Преимущества новой структуры

1. **Модульность**: Каждый модуль отвечает за свою область функциональности
2. **Читаемость**: Легче найти нужную функцию
3. **Поддержка**: Изменения в одном модуле не влияют на другие
4. **Переиспользование**: Общие функции в utils.js используются всеми модулями
5. **Масштабируемость**: Легко добавлять новые модули

## Зависимости между модулями

```
utils.js (базовые функции)
    ↓
socket.js (WebSocket, навигация)
    ↓
menu.js, users.js, purchase.js, dishes.js, reports.js, settings.js
(используют socket и utils)
```

## Обратная совместимость

Старый `config.js` сохранён как `config.js.old` для возможности отката.
